# Smart Lamp — умная лампа на ESP32 с управлением через MQTT

Проект реализует умную лампу на базе микроконтроллера **ESP32-C3** с WS2812-светодиодом. Устройство подключается к домашней Wi-Fi сети, получает команды через публичный MQTT-брокер и меняет своё поведение в реальном времени. Для первоначальной настройки не требуется ни мобильного приложения, ни дополнительного оборудования — достаточно смартфона или ноутбука.

В репозитории два независимых компонента:

- `main.c` — прошивка для ESP32 (ESP-IDF, C)
- `main.py` — управляющий сценарий на Python, запускается на любом компьютере

## Архитектура системы

```
┌──────────────────────────────────────────────────┐
│                     Python-PC                    │
│  main.py → публикует ON/OFF в MQTT-топик         │
└─────────────────────┬────────────────────────────┘
                      │ TCP/1883
                      ▼
          ┌───────────────────────┐
          │  broker.hivemq.com    │  (публичный MQTT-брокер)
          └───────────┬───────────┘
                      │ TCP/1883
                      ▼
┌──────────────────────────────────────────────────┐
│                    ESP32-C3                      │
│  Wi-Fi STA ← подключён к роутеру                │
│  MQTT клиент ← подписан на топик                │
│  WS2812 LED ← меняет цвет по команде            │
└──────────────────────────────────────────────────┘
```

Устройство и управляющий скрипт общаются исключительно через MQTT. Это позволяет запускать `main.py` откуда угодно — топик служит уникальным адресом конкретной лампы.

## Прошивка ESP32 (`main.c`)

### Первоначальная настройка Wi-Fi

При первом включении ESP32 не знает, к какому роутеру подключаться. Чтобы не зашивать SSID и пароль в код, устройство использует режим **APSTA**: одновременно поднимает собственную точку доступа и пытается подключиться как клиент.

| Параметр | Значение |
|---|---|
| Имя AP | `ESP32_Setup_Device` |
| Пароль AP | `12345678` |
| IP устройства в режиме AP | `192.168.4.1` |

После подключения к точке доступа `ESP32_Setup_Device` нужно открыть браузер и перейти по адресу `http://192.168.4.1`. Откроется минималистичная HTML-форма, в которую вводятся SSID и пароль домашней сети. По нажатию кнопки «Connect» данные уходят GET-запросом на `/config`, сохраняются в энергонезависимой памяти NVS и ESP32 тут же пробует подключиться к указанной сети.

При последующих перезагрузках устройство читает сохранённые данные и подключается к роутеру автоматически, без показа формы. Если Wi-Fi недоступен — AP остаётся активной и форма всегда доступна для повторной настройки.

### Подключение к MQTT

Как только ESP32 получает IP-адрес от роутера, запускается MQTT-клиент. Брокер — публичный `broker.hivemq.com`, топик задаётся в константе `device_topic`:

```c
static char device_topic[64] = "secrethashlamp8126371726369213697/default/power";
```

Длинный уникальный префикс топика (`secrethashlamp8126371726369213697`) нужен для того, чтобы исключить случайные пересечения с другими пользователями публичного брокера. При необходимости топик можно изменить, скомпилировав прошивку заново.

После успешного подключения к брокеру ESP32 подписывается на свой топик и ждёт сообщений `ON` или `OFF`.

### Логика светодиода

Единственный WS2812-пиксель выполняет сразу две роли: показывает состояние лампы и индицирует статус сетевых подключений. Управление пикселем вынесено в отдельную FreeRTOS-задачу `led_task`, которая постоянно крутится в цикле.

Когда лампа включена командой `ON` — горит **белый** (RGB 255, 255, 255).

Когда лампа выключена — цвет и паттерн мигания отражают текущий статус соединения:

| Режим | Цвет | Паттерн | Означает |
|---|---|---|---|
| `LED_MODE_SLOW_BLINK` | 🔵 Синий | 1 Гц (1 с вкл / 1 с выкл) | Нет Wi-Fi |
| `LED_MODE_FAST_BLINK` | 🔴 Красный | 5 Гц (0.2 с вкл / 0.2 с выкл) | Wi-Fi есть, нет MQTT |
| `LED_MODE_DOUBLE_BLINK` | 🟢 Зелёный | Двойная вспышка, пауза 1.5 с | Всё подключено |

Переключение между режимами происходит в обработчиках событий Wi-Fi и MQTT — без дополнительных таймеров или опроса состояния.

### Обработка URL-параметров

Стандартный HTTP-запрос с формы браузера кодирует спецсимволы в строке запроса (например, пробел становится `%20`, символ `@` — `%40`). Функция `url_decode` самостоятельно декодирует такие последовательности, чтобы пароли с любыми символами сохранялись корректно:

```c
void url_decode(char *dst, const char *src) {
    // %XX → символ с кодом 0xXX
    // '+' → пробел
}
```

## Управляющий сценарий Python (`main.py`)

Скрипт запускается на любом компьютере с Python 3 и реализует **автоматический сценарий включения/выключения** лампы по расписанию, привязанному к системным секундам.

### Установка зависимостей

```bash
pip install paho-mqtt questionary
```

### Запуск

```bash
python main.py
```

При старте скрипт спросит ID (топик) лампы. По умолчанию подставляется тот же топик, что вшит в прошивку.

### Алгоритм сценария

Сценарий работает непрерывно и управляет лампой, ориентируясь на текущую секунду системных часов (0–59). Каждую минуту определяется «окно включения» — симметричный интервал вокруг 30-й секунды.

```
Текущая длительность = 20 сек → окно ON: [20, 40) секунды
Текущая длительность = 18 сек → окно ON: [21, 39) секунды
...
Текущая длительность = 10 сек → окно ON: [25, 35) секунды → сброс до 20
```

Каждую минуту длительность уменьшается на 1 секунду. Когда она опускается ниже 10 секунд — сбрасывается обратно до 20. Таким образом, сценарий циклически «сжимает» и «восстанавливает» период включения лампы.

Сообщение в MQTT публикуется **только при смене состояния** (переходе ON→OFF или OFF→ON), а не каждую итерацию. Это снижает нагрузку на брокер и уменьшает лишние срабатывания на стороне ESP32. Флаг `retain=True` гарантирует, что новое устройство при подключении сразу получит актуальное состояние.

При нажатии `Ctrl+C` скрипт корректно завершает работу: отправляет команду `OFF` и отключается от брокера.

### Пример вывода

```
Сценарий запущен для топика: secrethashlamp8126371726369213697/default/power
Нажмите Ctrl+C для выхода.
[12:00:00] >>> СЕТЬ: Отправлено ON
[12:00:00] Светильник ВКЛ | Интервал: 20.0-40.0 сек.
...
[12:00:40] >>> СЕТЬ: Отправлено OFF
[12:00:40] Светильник ВЫКЛ

--- Новая минута. Установлена длительность: 19 сек. ---
```

## Схема взаимодействия компонентов

```
app_main()
│
├── nvs_flash_init()           ← инициализация хранилища
├── led_strip_new_rmt_device() ← инициализация WS2812
├── xTaskCreate(led_task)      ← запуск задачи мигания
├── esp_wifi_init()            ← инициализация Wi-Fi
│
├── esp_wifi_set_mode(APSTA)   ← AP для настройки + STA для роутера
├── esp_wifi_start()
├── esp_wifi_connect()         ← попытка подключения (если есть NVS-данные)
└── start_webserver()          ← HTTP-сервер для формы настройки
         │
         └── /config handler
               ├── url_decode()
               ├── save_wifi_credentials() → NVS
               └── esp_wifi_connect()      → event_handler()
                                                    │
                                         IP_EVENT_STA_GOT_IP
                                                    │
                                         esp_mqtt_client_start()
                                                    │
                                         MQTT_EVENT_CONNECTED
                                                    │
                                         subscribe(device_topic)
                                                    │
                                         MQTT_EVENT_DATA
                                                    │
                                         lamp_state = true/false
```

## Требования и зависимости

### Прошивка (ESP-IDF)

- ESP-IDF v5.x
- Целевой чип: ESP32-C3 (или совместимый с GPIO 8 для WS2812)
- Компоненты: `led_strip`, `mqtt_client`, `esp_http_server`, `nvs_flash`

### Python-скрипт

| Пакет | Назначение |
|---|---|
| `paho-mqtt >= 2.0` | MQTT-клиент (используется CallbackAPIVersion.VERSION2) |
| `questionary` | Интерактивный ввод топика в терминале |

## Безопасность и ограничения

- **Публичный брокер.** `broker.hivemq.com` открыт для всех — любой человек, знающий топик, может управлять лампой. Для продакшн-использования рекомендуется приватный брокер с аутентификацией.
- **HTTP без шифрования.** Форма настройки передаёт SSID и пароль в открытом виде через GET-запрос. Поскольку это происходит внутри изолированной AP-сети ESP32, риск перехвата минимален, но для критичных применений следует рассмотреть HTTPS.
- **Пароль AP жёстко задан.** `BOX_WIFI_PASS = "12345678"` — пароль от точки доступа для настройки нужно поменять перед развёртыванием.
- **Сброс настроек.** Очистить сохранённые Wi-Fi-данные можно вызовом `nvs_flash_erase()` при старте (с последующим `nvs_flash_init()`), либо через стандартный механизм Factory Reset в ESP-IDF.
