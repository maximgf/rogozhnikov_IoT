## Обзор системы

### Назначение
Система реализует полудуплексную (half-duplex) связь между двумя микроконтроллерами Arduino с использованием азбуки Морзе. Каждый МК может выступать как передатчиком, так и приёмником, но не одновременно.

### Архитектура
```
МК1 (TX) ────GPIO───> МК2 (RX)
             PIN_TX → PIN_RX
             
МК2 (TX) ────GPIO───> МК1 (RX)  
             PIN_TX → PIN_RX
```

### Основные компоненты

**Передатчик (TX):**
- Кольцевой буфер для накопления символов
- Конечный автомат для управления временными интервалами
- Преобразование ASCII → Морзе
- Протокол обрамления (Start/End маркеры)

**Приёмник (RX):**
- Прерывание по изменению уровня на GPIO
- Измерение длительности импульсов
- Декодирование Морзе → ASCII
- Обнаружение пауз между символами/словами

**Индикация:**
- 7-сегментный дисплей через 74HC595 (отображение принятых символов)
- Светодиод на PIN_LED (индикация передачи)

## Временные диаграммы

### 1. Базовые временные единицы

```
TIME_UNIT = 100 ms (1t)

Точка (.):     ████ = 100 ms (1t)
Тире (-):      ████████████ = 300 ms (3t)
Пауза внутри:  ░░░░ = 100 ms (между точками/тире в букве)
Пауза буквы:   ░░░░░░░░░░░░ = 300 ms (между буквами)
Пауза слова:   ░░░░░░░░░░░░░░░░░░░░░░░░ = 700 ms (между словами)
```

### 2. Передача точки (DOT)

```
TIME:    0ms      100ms     200ms
        ┌──────────┐
PIN_TX: │   HIGH   │  LOW
        └──────────┘
         DOT_TIME   SYMBOL_GAP
        (100ms)     (100ms)
```

### 3. Передача тире (DASH)

```
TIME:    0ms          300ms     400ms
        ┌──────────────────────┐
PIN_TX: │       HIGH           │  LOW
        └──────────────────────┘
              DASH_TIME         SYMBOL_GAP
              (300ms)           (100ms)
```

### 4. Передача буквы 'A' (• −)

```
TIME:    0     100   200   300   400   500   600   700   800
        ┌────┐     ┌──────────────┐     
PIN_TX: │ .  │ gap │      -       │ gap │    LETTER_GAP
        └────┘     └──────────────┘     │ (300ms до след. буквы)
         100ms 100ms    300ms     100ms
```

### 5. Полный кадр передачи "AB "

```
┌─ START MARKER "-.-.-" ─┐  ┌─ 'A' ".-" ─┐  ┌─ 'B' "-..." ─┐  WORD_GAP  ┌─ END MARKER "...-.-" ─┐

███│░│█████│░│███│LETTER_GAP│███│░│█████│░│███│░│███│LETTER_GAP│███████│███│░│█████│░│███░░░░░░░░░░░│███│░│███│░│███│░│███│░│███
-  │ │  -  │ │ - │          │ . │ │  -  │          │-  │ │ . │ │ . │ │ .          │               │ . │ │ . │ │ . │ │ - │ │ . │ │ -

START (KA)                   A                      B                     SPACE (700ms)             END (SK)
```

### 6. Приём импульса с измерением

```
PIN_RX:  ────┐         ┌────────
             │ HIGH    │ 
             └─────────┘
              ↑         ↑
         rxPulseStart  ISR triggers
                       rxPulseWidth = now - rxPulseStart
                       
Анализ:
- Если width ≈ 100ms (±50ms) → точка "."
- Если width ≈ 300ms (≥250ms) → тире "-"
- Иначе → шум (игнорируется)
```

### 7. Определение границ символов

```
Входящий сигнал:
███│░│███████│░░░░░░░░░░░░░░░░░░░░░░░░│███│░│███████
 .  │ │   -   │  PAUSE > LETTER_GAP   │ .  │ │   -
 
    └──── буква 'A' ────┘              └─── новая буква

Детектирование:
- Время с последнего края (rxLastEdge) > LETTER_GAP (300ms)
- → Декодировать накопленную последовательность
- → Очистить rxBufferSeq
```

## Конечные автоматы

### 1. Автомат передатчика (TX FSM)

```
┌──────────────────────────────────────────────────────────────┐
│                         TX_IDLE                              │
│  [condition] txBuffer не пуст (peekTxBuffer() != 0)         │
│  [state_change] → TX_SEND_START                             │
│  [action] Готовность к отправке кадра                       │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│                      TX_SEND_START                          │
│  [action] currentMorseSeq = PROTOCOL_START ("-.-.-")        │
│  [action] txSeqIndex = 0                                     │
│  [action] txReturnState = TX_SEND_PAYLOAD                   │
│  [state_change] → TX_SIGNAL_HIGH                            │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│                     TX_SIGNAL_HIGH                          │
│  [condition] if txSeqIndex >= currentMorseSeq.length()      │
│  [state_change] → TX_NEXT_CHAR_WAIT                         │
│  [action] txLastTime = millis()                             │
│                                                              │
│  [condition] else (есть ещё символы)                        │
│  [action] signal = currentMorseSeq[txSeqIndex]              │
│  [action] digitalWrite(PIN_TX, HIGH)                        │
│  [action] digitalWrite(PIN_LED, HIGH)                       │
│  [state_change] → TX_WAIT_HIGH                              │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│                      TX_WAIT_HIGH                           │
│  [condition] if (millis() - txLastTime) >= duration         │
│       где duration = DOT_TIME или DASH_TIME                 │
│  [action] digitalWrite(PIN_TX, LOW)                         │
│  [action] digitalWrite(PIN_LED, LOW)                        │
│  [action] txLastTime = millis()                             │
│  [state_change] → TX_WAIT_LOW                               │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│                       TX_WAIT_LOW                           │
│  [condition] if (millis() - txLastTime) >= SYMBOL_GAP      │
│  [action] txSeqIndex++                                       │
│  [state_change] → TX_SIGNAL_HIGH                            │
│       (возврат к следующему элементу символа)               │
└──────────────────────────────────────────────────────────────┘
                            ↓ (после всех элементов)
┌──────────────────────────────────────────────────────────────┐
│                    TX_NEXT_CHAR_WAIT                        │
│  [condition] if (millis() - txLastTime) >= LETTER_GAP      │
│  [state_change] → txReturnState                             │
│       (TX_SEND_PAYLOAD или TX_IDLE)                         │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│                     TX_SEND_PAYLOAD                         │
│  [condition] if peekTxBuffer() == 0 (буфер пуст)           │
│  [state_change] → TX_SEND_END                               │
│                                                              │
│  [condition] if peekTxBuffer() == ' ' (пробел)             │
│  [action] popTxBuffer()                                      │
│  [action] txLastTime = millis()                             │
│  [state_change] → TX_NEXT_CHAR_WAIT (пауза WORD_GAP)       │
│                                                              │
│  [condition] else (обычный символ)                          │
│  [action] currentMorseSeq = charToMorse(c)                  │
│  [action] popTxBuffer()                                      │
│  [action] txReturnState = TX_SEND_PAYLOAD                   │
│  [state_change] → TX_SIGNAL_HIGH                            │
└──────────────────────────────────────────────────────────────┘
                            ↓ (после всех символов)
┌──────────────────────────────────────────────────────────────┐
│                       TX_SEND_END                           │
│  [action] currentMorseSeq = PROTOCOL_END ("...-.-")         │
│  [action] txSeqIndex = 0                                     │
│  [action] txReturnState = TX_IDLE                           │
│  [state_change] → TX_SIGNAL_HIGH                            │
└──────────────────────────────────────────────────────────────┘
                            ↓
                       (TX_IDLE)
```

### 2. Автомат приёмника (RX FSM)

Приёмник работает комбинированно: через прерывания (ISR) и основной цикл.

#### 2.1. Обработчик прерывания (rxISR)

```
┌──────────────────────────────────────────────────────────────┐
│                   ISR: Изменение PIN_RX                     │
│  [condition] if digitalRead(PIN_RX) == HIGH                 │
│  [action] rxPulseStart = millis()                            │
│  [action] rxLastEdge = millis()                              │
│       (начало импульса HIGH)                                 │
│                                                              │
│  [condition] if digitalRead(PIN_RX) == LOW                  │
│  [action] rxPulseWidth = millis() - rxPulseStart            │
│  [action] rxPulseReady = true                                │
│  [action] rxLastEdge = millis()                              │
│       (конец импульса HIGH, данные готовы)                  │
└──────────────────────────────────────────────────────────────┘
```

#### 2.2. Основной цикл приёмника (runReceiverFSM)

```
┌──────────────────────────────────────────────────────────────┐
│                  Обработка принятого импульса               │
│  [condition] if rxPulseReady == true                        │
│  [action] width = rxPulseWidth                               │
│  [action] rxPulseReady = false                               │
│                                                              │
│  [condition] if width ≈ DOT_TIME (100ms ± 50ms)            │
│  [action] rxBufferSeq += "."                                 │
│                                                              │
│  [condition] if width ≈ DASH_TIME (≥ 250ms)                │
│  [action] rxBufferSeq += "-"                                 │
│                                                              │
│  [condition] else (не подходит ни под что)                  │
│  [action] игнорировать (шум)                                 │
│                                                              │
│  [action] rxLastActivity = millis()                          │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│               Обнаружение границ символов                   │
│  [condition] if PIN_RX == LOW AND rxBufferSeq не пуст      │
│  [condition] AND (millis() - rxLastEdge) > LETTER_GAP      │
│  [action] decodeMorseChar(rxBufferSeq)                       │
│  [action] rxBufferSeq = ""                                   │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│                   decodeMorseChar(seq)                      │
│  [condition] if seq == PROTOCOL_START ("-.-.-")             │
│  [action] rxFrameActive = true                               │
│  [action] Serial.println("[RX] Frame START")                │
│  [state_change] начало приёма кадра                         │
│                                                              │
│  [condition] if seq == PROTOCOL_END ("...-.-")              │
│  [action] rxFrameActive = false                              │
│  [action] Serial.println("[RX] Frame END")                  │
│  [state_change] конец приёма кадра                          │
│                                                              │
│  [condition] if NOT rxFrameActive                           │
│  [action] return (игнорировать вне кадра)                   │
│                                                              │
│  [condition] if seq найдена в MORSE_LETTERS                │
│  [action] decoded = 'A' + index                              │
│  [action] displayChar(decoded)                               │
│  [action] Serial.print(decoded)                              │
│                                                              │
│  [condition] if seq найдена в MORSE_NUMBERS                │
│  [action] decoded = '0' + index                              │
│  [action] displayChar(decoded)                               │
│  [action] Serial.print(decoded)                              │
│                                                              │
│  [condition] else (не распознано)                           │
│  [action] игнорировать                                       │
└──────────────────────────────────────────────────────────────┘
```

### 3. Обработка ввода с Serial (handleSerialInput)

```
┌──────────────────────────────────────────────────────────────┐
│                    handleSerialInput()                      │
│  [condition] while Serial.available() > 0                   │
│  [action] c = Serial.read()                                  │
│  [action] c = toupper(c)                                     │
│                                                              │
│  [condition] if (c in 'A'-'Z') OR (c in '0'-'9') OR c==' ' │
│  [condition] AND буфер не полон                             │
│  [action] txBuffer[txHead] = c                               │
│  [action] txHead = (txHead + 1) % 64                        │
│       (добавление в кольцевой буфер)                        │
│                                                              │
│  [condition] else (недопустимый символ)                     │
│  [action] игнорировать                                       │
└──────────────────────────────────────────────────────────────┘
```

## Режимы работы

Данная система по умолчанию работает в **одном режиме** (назовём его **"нормальный режим"**), но концептуально можно выделить два подхода к обработке сигналов:

### 1. Normal Mode (Нормальный режим) — Реализовано

**Характеристики:**
- Используется протокол обрамления (START/END маркеры)
- Приёмник игнорирует данные вне кадра (rxFrameActive)
- Декодирование происходит только между маркерами
- Обеспечивает синхронизацию и защиту от случайного шума
- Гарантирует, что приёмник интерпретирует только намеренно отправленные данные

**Процесс:**
```
TX отправляет: START → Данные → END
RX принимает:  [игнор] → START обнаружен → rxFrameActive=true → 
               декодирование данных → END обнаружен → rxFrameActive=false → [игнор]
```

**Преимущества:**
- Защита от ложных срабатываний на шум
- Чёткое разграничение кадров
- Возможность передачи нескольких сообщений последовательно
- Детектирование потерянных START/END маркеров

**Недостатки:**
- Дополнительные накладные расходы (служебные символы)
- Задержка перед началом декодирования полезной нагрузки

### 2. Raw Mode (Сырой режим) — Не реализовано, но возможно

**Концепция:**
- Приёмник декодирует все входящие сигналы без проверки маркеров
- Не используется флаг rxFrameActive
- Любая последовательность точек/тире интерпретируется как символ

**Как бы это выглядело:**
```cpp
void decodeMorseChar(String seq) {
  // Без проверки START/END
  // if (seq == PROTOCOL_START || seq == PROTOCOL_END) return;
  
  // Сразу декодируем
  for (int i = 0; i < 26; i++) {
    if (seq == MORSE_LETTERS[i]) {
      char decoded = 'A' + i;
      displayChar(decoded);
      Serial.print(decoded);
      return;
    }
  }
  // ... аналогично для цифр
}
```

**Преимущества:**
- Нулевые накладные расходы
- Мгновенное начало декодирования
- Простота реализации

**Недостатки:**
- Любой шум на линии может быть ошибочно интерпретирован как символ
- Нет механизма синхронизации
- Невозможно определить начало/конец сообщения
- Возможны ложные срабатывания при наводках

### Сравнительная таблица

| Параметр | Normal Mode (текущий) | Raw Mode (гипотетический) |
|----------|----------------------|---------------------------|
| Обрамление | START/END маркеры | Нет |
| Защита от шума | Высокая | Низкая |
| Накладные расходы | ~10 символов Морзе | 0 |
| Синхронизация | Есть | Нет |
| Сложность | Средняя | Низкая |
| Надёжность | Высокая | Низкая |
| Применимость | Зашумлённая среда | Идеальная линия |

## Проектные решения

### 1. Использование конечных автоматов (FSM)

**Решение:** Логика передатчика реализована через enum-based state machine с 8 состояниями.

**Обоснование:**
- **Читаемость:** Явное указание текущего состояния упрощает отладку
- **Расширяемость:** Легко добавить новые состояния (например, TX_ERROR)
- **Детерминированность:** Гарантирует предсказуемые переходы
- **Временная точность:** Каждое состояние отвечает за конкретный временной интервал

**Альтернативы:**
- Вложенные `if-else` (сложнее отслеживать состояния)
- Callback-функции (избыточная сложность для простой логики)

### 2. Прерывания (Interrupts) для приёма

**Решение:** Используется `attachInterrupt(digitalPinToInterrupt(PIN_RX), rxISR, CHANGE)` для отслеживания фронтов сигнала.

**Обоснование:**
- **Точность:** Захват точного времени изменения уровня
- **Асинхронность:** Не пропускаем импульсы во время выполнения другого кода
- **Эффективность:** CPU не занят постоянным опросом (polling)

**Важное ограничение:**
```cpp
void rxISR() {
  // КРИТИЧНО: ISR должна быть максимально короткой!
  // Только чтение времени и установка флагов
  // NO: Serial.print(), delay(), долгие вычисления
}
```

### 3. Кольцевой буфер (Ring Buffer)

**Решение:** Передатчик использует циклический массив `txBuffer[64]` с указателями `txHead` и `txTail`.

**Обоснование:**
- **Эффективность памяти:** Фиксированный размер, нет динамического выделения
- **FIFO порядок:** Символы передаются в порядке ввода
- **Защита от переполнения:** Проверка `nextHead != txTail` предотвращает затирание данных

**Формула вместимости:**
```cpp
int nextHead = (txHead + 1) % 64;
// Эффективная вместимость = 63 символа (один слот резервируется для различения пустого/полного буфера)
```

### 4. Протокол обрамления

**Решение:** Используются стандартные сигналы Морзе:
- `START = "-.-.-"` (prosign KA — "начало передачи")
- `END = "...-.-"` (prosign SK — "конец работы")

**Обоснование:**
- **Стандартизация:** Соответствует реальным радиолюбительским протоколам
- **Однозначность:** Маркеры не совпадают с обычными буквами/цифрами
- **Синхронизация:** Приёмник знает, когда начинать и заканчивать декодирование

### 5. Допуски (Tolerance)

**Решение:** `TOLERANCE = 50ms` для распознавания точек и тире.

**Обоснование:**
- **Jitter таймеров:** `millis()` имеет погрешность
- **Задержки ISR:** Прерывания могут быть отложены другими ISR
- **Производственные отклонения:** Разброс характеристик кварцевых резонаторов

**Диапазоны детектирования:**
```cpp
// DOT: 50ms < width < 250ms
if (width > (DOT_TIME - TOLERANCE) && width < (DOT_TIME + TOLERANCE + 50))

// DASH: width > 250ms
if (width > (DASH_TIME - TOLERANCE))
```

### 6. Блокировка вне кадра (rxFrameActive)

**Решение:** Флаг `rxFrameActive` включается по START и выключается по END.

**Обоснование:**
- **Фильтрация шума:** Игнорируются случайные импульсы до начала кадра
- **Экономия ресурсов:** Не тратится время на декодирование мусора
- **Чёткая семантика:** Пользователь видит границы сообщения в Serial

### 7. 7-сегментный дисплей через 74HC595

**Решение:** Используется сдвиговый регистр для управления дисплеем с 3 GPIO пинами.

**Обоснование:**
- **Экономия пинов:** 3 вместо 8 для управления сегментами
- **Расширяемость:** Легко добавить больше разрядов (каскадирование)
- **Простота:** Стандартная функция `shiftOut()`

**Кодировка:**
```cpp
// Пример: '0' = 0x3F = 0b00111111
//   A
//  F B
//   G
//  E C
//   D  (DP не используется)
```

### 8. Обработка пробелов

**Решение:** При встрече `' '` в буфере не отправляется символ Морзе, а выжидается `WORD_GAP`.

**Обоснование:**
- **Соответствие стандарту:** В Морзе пробел = длинная пауза (7t)

## Ограничения системы

### 1. Односторонняя передача (полудуплекс)

**Описание:** Система не поддерживает одновременную передачу и приём.

**Причина:**
- Передатчик и приёмник работают независимо, но на одном МК
- Нет арбитража доступа к линии
- Конфликт при одновременной работе TX и RX на обоих МК приведёт к столкновению данных

**Сценарий проблемы:**
```
МК1: TX активен (отправляет "HELLO")
МК2: TX активен (отправляет "WORLD")
    → Линии перекрываются → Оба RX получают искажённые данные
```

**Решение (не реализовано):**
- CSMA/CD (Carrier Sense Multiple Access with Collision Detection)
- Token passing
- Мастер-слейв архитектура

### 2. Размер буфера

**Ограничение:** Максимум 63 символа в очереди передачи.

**Последствия:**
- Длинные сообщения (>63 символа) будут обрезаны
- Быстрый ввод может переполнить буфер

**Решение:**
```cpp
if (nextHead != txTail) { // Проверка переполнения
  txBuffer[txHead] = c;
  txHead = nextHead;
} // else символ просто отбрасывается
```

### 3. Точность временных интервалов

**Ограничение:** Функция `millis()` имеет разрешение 1-4 мс (зависит от МК).

**Последствия:**
- При TIME_UNIT = 100ms погрешность ~2%
- Накопление ошибки при длинных сообщениях
- Критично для высоких скоростей (если уменьшить TIME_UNIT)

**Возможные улучшения:**
- Использовать аппаратные таймеры (Timer1/Timer2)
- Микросекундное разрешение с `micros()` (но оно переполняется каждые ~70 минут)

### 4. Отсутствие коррекции ошибок

**Ограничение:** Нет механизма обнаружения/исправления ошибок.

**Последствия:**
- Искажённый импульс → неправильный символ
- Пропущенный START → вся передача игнорируется
- Помехи на линии → мусор в выводе

**Возможные решения:**
- CRC/checksum в конце кадра
- Подтверждения (ACK/NACK)
- Повторная передача (ARQ)

### 5. Чувствительность к шуму

**Ограничение:** При использовании без развязки (pull-up/pull-down резисторы) возможны ложные срабатывания.

**Рекомендации:**
```
PIN_RX ──────┤├─────────── Входной сигнал
             0.1µF
             (фильтр помех)
             
        10kΩ
PIN_RX ───┬───────── Сигнал
          │
         GND
        (pull-down)
```
