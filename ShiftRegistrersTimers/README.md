## 1. Общее описание
Программа реализует цифровой счетчик от 0 до 59. Данные выводятся на дисплей с использованием программного протокола, похожего на SPI (для работы со сдвиговыми регистрами типа 74HC595). Обновление дисплея и инкремент счетчика происходят раз в секунду по прерыванию таймера.

## 2. Конфигурация оборудования (Пины)
Используется прямой доступ к портам микроконтроллера ATmega328P (Arduino Uno).

| Макрос | Порт | Пин Arduino | Описание |
| :--- | :--- | :--- | :--- |
| **OUT_L_PORT** | PORTD5 | D5 | **Latch** (Защелка) — фиксирует данные в регистре |
| **OUT_C_PORT** | PORTD3 | D3 | **Clock** (Такты) — синхронизация передачи |
| **OUT_D_PORT** | PORTD7 | D7 | **Data** (Данные) — передача конкретного бита |

## 3. Глобальные переменные
* `volatile int t_count`: Основной счетчик секунд (0–59).
* `volatile int u_val`: Буфер для хранения нового значения, полученного из Serial. Значение `-1` означает отсутствие новых данных.
* `volatile bool active`: Флаг состояния. Становится `true` после первой установки времени.
* `const uint8_t MAP[10]`: Массив масок для формирования цифр 0–9 на семисегментном индикаторе.

## 4. Описание функций

### `void push_bit(uint8_t data)`
Программная реализация передачи 8-битного байта во внешний сдвиговый регистр.
* Перебор битов идет от старшего (MSB) к младшему (LSB).
* Для каждого бита устанавливается состояние на линии данных (`OUT_D`) и генерируется тактовый импульс на линии (`OUT_C`).



### `uint8_t get_hex(int n)`
Возвращает соответствующую маску из массива `MAP` для цифры `n`. Если число вне диапазона 0–9, возвращает 0.

### `void setup()`
1.  Настраивает пины L, C, D на выход.
2.  Инициализирует Serial-порт на скорости 9600 бод.
3.  **Настройка Таймера 1 (16-бит):**
    * Режим CTC (Clear Timer on Compare).
    * Предделитель 1024.
    * `OCR1A = 15624` — обеспечивает прерывание ровно 1 раз в секунду ($16 \text{ MHz} / (1024 \times (15624 + 1)) = 1 \text{ Hz}$).

### `ISR(TIMER1_COMPA_vect)`
Обработчик прерывания таймера (выполняется каждую секунду):
1.  **Синхронизация:** Считывает значение `u_val` в локальную переменную и сбрасывает его.
2.  **Логика счетчика:** * Если система не активна и получены данные — запускает счет.
    * Если активна — инкрементирует `t_count` (сброс после 59).
3.  **Отображение:** Разделяет число на десятки и единицы, отправляет их через `push_bit` и "защелкивает" результат на индикаторе.

### `void loop()`
Отвечает за прием данных по Serial:
1.  Накапливает до 2-х цифр в буфер `b`.
2.  Если ввод прекратился (пауза более 25 мс), преобразует строку в число и безопасно (с запретом прерываний) записывает его в `u_val`.

## 5. Алгоритм работы пользователя
1.  После прошивки дисплей может не показывать динамику (пока `active = false`).
2.  Отправьте в монитор порта число (например, `42`).
3.  Таймер примет это значение, отобразит его и начнет ежесекундный отсчет.
